# PRD.md - 产品需求文档

## 1. 业务目标

### 1.1 核心价值

解决嵌入式/Android 开发中物理开发板串口共享难题：
- **痛点**：开发板通常连接在特定 PC，其他工程师无法直接查看 Log
- **方案**：通过网络虚拟化串口，实现多人同时读取、有序写入

### 1.2 目标用户

| 角色 | 场景 | 核心诉求 |
|------|------|----------|
| 嵌入式工程师 | 调试开发板 | 无需移动即可查看远程 Log |
| Android 系统工程师 | 内核调试 | 兼容现有工具（minicom/picocom） |
| 测试工程师 | 自动化测试 | 虚拟串口可被脚本调用 |

## 2. 用户故事

### US-001: 服务端启动串口共享

```
As 嵌入式工程师
I want 在连接开发板的机器上启动服务端，自动扫描并共享所有串口
So that 局域网内其他工程师可以访问
```

**验收标准：**
- [ ] 自动扫描 `/dev/ttyUSB*`、`/dev/ttyACM*`
- [ ] 固定波特率 115200
- [ ] 配置文件指定监听端口

### US-002: 客户端配置连接

```
As 远程工程师
I want 通过配置文件连接远程串口服务
So that 快速建立连接无需复杂操作
```

**验收标准：**
- [ ] 配置文件指定服务器 IP/端口
- [ ] 一键启动连接
- [ ] 自动创建虚拟串口

### US-007: AI 调用 CLI 接口

```
As AI Agent（如 Claude Code）
I want 通过 CLI 命令直接获取远程串口 Log 数据
So that 可以在对话中实时分析 Log，无需用户手动复制
```

**验收标准：**
- [ ] 提供 `shareserial log [options]` 命令
- [ ] 支持输出到 stdout（便于 AI 程序化读取）
- [ ] 支持过滤关键词（如 `shareserial log --filter "ERROR"`）
- [ ] 支持时间范围（如 `shareserial log --since "5m"`）
- [ ] 支持 Skill 封装（Claude Code 可直接调用）
- [ ] 输出格式支持 JSON（便于程序解析）

### US-003: 客户端创建虚拟串口

```
As 工程师
I want 连接后本地出现标准虚拟串口（/dev/vttyShare0）
So that 现有工具（minicom/picocom）无需修改即可使用
```

**验收标准：**
- [ ] 创建 POSIX PTY 虚拟串口
- [ ] 兼容 termios 标准
- [ ] 支持常用波特率（9600 ~ 1500000）

### US-004: 多人同时读取 Log

```
As 多位工程师
I want 同时读取同一串口的 Log 输出
So that 各自独立调试不影响
```

**验收标准：**
- [ ] 服务端 One-to-Many 广播
- [ ] 单客户端网络卡顿不影响其他客户端
- [ ] 延迟 < 5ms

### US-005: 写入仲裁（独占模式）

```
As 工程师
I want 申请写锁后独占串口写入权限
So that 避免多人同时输入导致乱码
```

**验收标准：**
- [ ] 默认只读权限
- [ ] 支持申请/释放写锁
- [ ] 写锁超时自动释放（可配置，默认 30s）
- [ ] 写锁状态广播通知所有客户端

### US-006: 断线自动重连

```
As 工程师
I want 网络波动时客户端自动重连
So that 上层 Log 工具不中断
```

**验收标准：**
- [ ] 检测网络断开
- [ ] 自动重试连接
- [ ] 重连后恢复 Log 输出
- [ ] 虚拟串口设备保持存在

## 4. 非功能性需求

### 4.1 性能指标

| 指标 | 目标 |
|------|------|
| 网络延迟 | < 10ms |
| 波特率 | 固定 115200（Phase 1） |
| 并发客户端 | ≥ 5 |
| 24h 稳定性 | 无断流、无丢包、无内存泄漏 |

### 4.2 部署要求

- **Linux**: 单二进制文件，无依赖
- **权限**: 服务端需要串口访问权限（dialout 组）
- **配置**: 配置文件指定服务器 IP/端口，无需 mDNS

## 5. 排除范围（Phase 1）

- Windows 平台支持（Phase 2/3）
- 加密传输（后续可添加 TLS）
- 用户认证（后续可添加 Token）
- GUI 界面（Phase 2 Windows 客户端）
